# Hire_X - Authentication Flow Diagnostic Report

This report analyzes the synchronization and functionality gaps between the React frontend and the Node.js/Express backend concerning user authentication (Register, Login, and Logout).

---

## 1️⃣ Register Flow Analysis

**Frontend status (`client/src/features/auth/Register/Register.jsx`):**
- **Fields collected:** `fullName`, `username`, `email`, `password`, `confirmPassword`, `role`, `agreeToTerms`.
- **API implementation:** Uses `@tanstack/react-query` to make an Axios `POST` request to `http://localhost:5000/api/auth/register`.
- **Transformation:** Properly passes `selectedRole` mapping to either `"jobseeker"` or `"employer"`.
- **Response handling:** Currently, it successfully resolves and triggers `console.log("Registration successful:", data)`. It does NOT save the returned JWT token to `localStorage` or cookies, and it does NOT redirect the user to the login or dashboard pages upon success.

**Backend status (`server/src/controllers/auth.controller.js`):**
- **Validation:** Enforces `fullName`, `username`, `email`, `password`, and `role`.
- **Logic:** Successfully checks for uniqueness, hashes the password via `bcrypt`, generates a 7-day `jsonwebtoken` (`JWT_SECRET`), and seeds either an empty `JobSeeker` or `Recruiter` profile depending on the `role`.
- **Response Structure:** `{ success: true, message, token, user }`.

**Conclusion:** The register endpoint paths and primary fields match correctly. However, the frontend lacks post-registration business logic (saving state, redirecting).

---

## 2️⃣ Login Flow Analysis

**Frontend status (`client/src/features/auth/Login/Login.jsx`):**
- **Fields collected:** `email`, `password`, `remember` (checkbox).
- **API implementation:** **MISSING.** Currently, the `onSubmit` handler only runs `console.log("Sign in:", data);`. There is no API request built to communicate with the backend.
- **Token handling:** Missing entirely.
- **Redirection logic:** Missing entirely.

**Backend status (`server/src/controllers/auth.controller.js`):**
- **Validation:** Requires `email` and `password`.
- **Logic:** Correctly fetches the user via email, decrypts and compares the password via `bcrypt.compare`, and generates a JWT.
- **Response Structure:** `{ success: true, message, token, user }`.

**Conclusion:** The backend `/api/auth/login` is fully functional and ready to be consumed, but the frontend connection is completely unwritten.

---

## 3️⃣ Logout Flow Analysis

**Frontend status:**
- **UI & Logic:** **MISSING.** There is no dedicated logout button logic, nor any component discovered that clears the stored JWT token or redirects the user back to the public homepage.

**Backend status (`server/src/controllers/auth.controller.js`):**
- **API:** An endpoint (`POST /api/auth/logout`) exists and gracefully returns a HTTP 200 payload mapping `{ success: true, message: "Logged out successfully" }`.

---

## 4️⃣ Detected Issues

1. **Missing Frontend Login API Request:** `Login.jsx` is only a visual shell. It does not send POST requests to the backend API.
2. **Missing Token Storage:** Neither the Register nor Login frontend flows save the backend-supplied JWT token to `localStorage` or `sessionStorage`.
3. **Missing Authentication State:** The frontend lacks a Context Provider or Redux slice to track whether the user is globally "logged in," which is necessary for protected UI routing.
4. **Missing Axios Interceptors:** Protected backend routes utilizing `authMiddleware` require an `Authorization: Bearer <TOKEN>` header. The frontend presently has no mechanism attached to dynamically append this to future requests.
5. **No Post-Action Redirection:** Successful `/register` or expected `/login` requests don't redirect to the user's dashboard utilizing React Router's `useNavigate`.
6. **No Logout Implementation:** There is no logic mapped in the frontend to wipe local credentials.

---

## 5️⃣ Suggested Fixes

### Frontend Fixes:
1. **Implement `useMutation` in Login:** Build out the `useMutation` hook in `Login.jsx` identically to `Register.jsx`, targeting `http://localhost:5000/api/auth/login`.
2. **Handle the JWT Token:** Upon `onSuccess` in both `Register.jsx` and `Login.jsx`, immediately store the token via `localStorage.setItem("token", data.token)`.
3. **Handle Current User State:** Create an `AuthContext.jsx` inside `client/src/context/` to hydrate global authentication states securely.
4. **Build an Axios Interceptor:** Either build a utility file (e.g., `client/src/utils/axiosInstance.js`) configuring axios to append `Authorization: Bearer ${localStorage.getItem('token')}` across all requests, or explicitly fetch the token manually.
5. **Add Redirects:** Use `useNavigate` from `react-router-dom` on successful authentication resolutions (e.g., `nav("/dashboard")` or `nav("/login")`).
6. **Implement Logout:** Attach an `onClick` parameter locally triggering `localStorage.removeItem("token")` followed by `nav("/")` to effectively "log out" the user statelessly.

### Backend Fixes:
The backend logic operates robustly. No fixes are mandatory, though implementing HTTP-only Cookies for JWT storage could replace `localStorage` transmission patterns to mitigate XSS vector threats if heightened security is desired later.
