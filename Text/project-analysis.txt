5. LAYOUT / STRUCTURE ISSUES (NON-CSS)
================================================================================

• Feature folder duplication: Overlapping paths (e.g. jobSeeker Companies vs public Companies) are intentional for roles but increase confusion; same for possible duplicate MyApplications paths — risk of editing stale copies.

• Route naming inconsistency: Mixed casing (e.g. /jobSeeker/EditProfile vs lowercase paths) — harder to remember and document.

• “shared/Chat” under features while Messages wrappers are thin re-exports — good separation, but the folder name “shared” is easy to miss when onboarding.

• Large ChatPage.jsx: Many responsibilities (inbox, modals, block list, mutations) — harder to debug for a semester project (acceptable but dense).

• Backend route order: Dynamic routes (e.g. /:id) must stay after static segments (already a common Express pattern; worth double-checking when adding new routes).

• Build artifact: client/dist committed or present in workspace can bloat repo — optional .gitignore improvement.

WHY THESE ARE “NOT IDEAL”
------------------------
They do not break the app by themselves, but they increase cognitive load, duplicate maintenance, and make deployment or grading setup error-prone without clear documentation.
